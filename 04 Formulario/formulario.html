Todos los formularios en la web, manejan métodos para el envio de las diferenes peticiones que realiza el usuario considerando la arquitectura cliente-servidor el manejo de las peticiones se hace por medio del protocolo http y actualmente por https, cada peticion debe de se atendida en un tiempo de respuesta conforme al modelo TCP. Los métodos principales de los formularios son :

    • GET :

      Este metodo se encarga de enviar la información del forulario por medio de la salida estándar (conocida como url ), del navegador, esto propicia que puedan existir inyecciones de código realizando violaciones a la seguridad del sistema.

      POST :

      Este método se encarga de enviar la información para poder registrar un cambio o un nuevo registro dentro del sistema enviandola igual, por el métood esándar pero mediante un ocultamiento, eso significa que no es vsible para el usuario lo que nos ofrece un mayor grado de sefguridad en el manejo de la información


    • En resumen, los metodos son :

    • Get : Sirve para enviar y realizar operaciones de lectura y de consulta
      Post: Sirve para enviar informacion de un recurso al servidor, con el fin de registrarlo
      Delete: sirve para eliminar la información de un recurso especifico dentro del servidor, teniendo como respuesta una actualización
      Head: Sirve para regresar el código de respuesta de cualquier cabecera HTTP asociada a un recurso
      Put : Sirve para almacenar la informacion de un recurso particular al conectarse al servicio para realizar la actualización
